# 用户反馈与贡献指南 / User Feedback & Contribution Guide

## 🤝 如何贡献 / How to Contribute

### 提交新案例 / Submit New Cases

如果你在使用中有新的案例，欢迎提交！

**案例格式**:
```markdown
### 案例标题 / Case Title

**背景 / Background**:
- 场景: [描述]
- 涉及方: [人员]
- 结果: [成功/失败]

**使用的 Skill**: [甩锅/背锅/砸锅]

**具体做法 / What I Did**:
[详细描述]

**效果 / Result**:
[结果如何]

**反思 / Reflection**:
[学到了什么]
```

### 提交话术改进 / Submit话术 Improvements

如果你觉得某些话术可以改进：

```markdown
**原话术 / Original**:
[原内容]

**建议改进 / Suggested Improvement**:
[改进内容]

**理由 / Reason**:
[为什么这样改更好]
```

### 报告问题 / Report Issues

如果你发现：
- 某些话术在实际使用中产生负面效果
- 有更适合的表达方式
- 需要补充更多使用场景

请提交 Issue，格式如下：

```markdown
**问题类型 / Issue Type**: [Bug/Improvement/Question]

**相关 Skill**: [甩锅/背锅/砸锅]

**问题描述 / Description**:
[详细描述]

**建议 / Suggestion**:
[你的建议]
```

## 📋 贡献流程 / Contribution Process

### 通过 GitHub 贡献 / Contribute via GitHub

1. **Fork 项目** / Fork the repository
2. **创建分支** / Create a branch
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **提交更改** / Commit your changes
   ```bash
   git commit -m "Add: your description"
   ```
4. **推送分支** / Push to the branch
   ```bash
   git push origin feature/your-feature-name
   ```
5. **创建 Pull Request** / Create a Pull Request

### 贡献规范 / Contribution Guidelines

- ✅ 保持双语（中文/英文）
- ✅ 遵循现有文档风格
- ✅ 确保内容基于真实职场逻辑
- ✅ 尊重他人隐私（案例中的敏感信息脱敏）
